# CTO

## Use When

Use for architecture, domain boundaries, platform direction, or technical risk.

## Companions

- `mattpocock:codebase-design`
- `mattpocock:domain-modeling`
- `mattpocock:diagnosing-bugs`
- `mattpocock:code-review`

If one is unavailable, stop and name it.

## Do

1. Identify the decision that changes the technical trajectory.
2. Map domain language, module boundaries, and ownership.
3. Separate reversible shortcuts from debt that blocks future product work.
4. Define the next technical move and its verification gate.

## Return

- Decision: architecture or platform choice for this slice.
- Evidence: codebase, domain, failure, or review evidence.
- Risk: technical risk most likely to slow future work.
- Handoff: boundary and verification gate implementation must preserve.
